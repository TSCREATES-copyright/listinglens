## 🔭 What Is ListingLens?

**ListingLens** is a fully browser-based, zero-backend resale pricing engine that helps everyday marketplace sellers make faster, more confident pricing decisions — without guessing, without spreadsheets, and without ever touching a server.

It answers one question that every resale seller asks every single time:

> *"What should I list this for so it actually sells — without leaving money on the table?"*

ListingLens is **not** a market scraper. It is **not** an AI tool. It is a **deterministic local pricing system** — fast, reliable, consistent, and entirely yours.

Every output is calculated from your inputs using proven resale heuristics. Same inputs. Same outputs. Every time.

## ✨ Core Features

### 🏷️ Deterministic Price Engine
Computes **list price**, **likely sell price**, **fast-sale price**, and **negotiation buffer** from your item details — using category anchors, condition multipliers, brand desirability weights, age decay curves, and urgency factors. No randomness. No AI drift. The same inputs always produce the same outputs.

### 🎯 Confidence Score
Measures how reliably the engine can price your item based on how complete and recognizable your inputs are. A high-confidence score means the output is tightly calibrated. A low-confidence score means you should add more detail — or accept a wider price range.

### 📋 Listing Title Builder
Generates a structured title pattern — not AI-generated copy, but a proven format:
`Brand + Item Type + Key Feature + Condition + Local Keyword`
Titles that are scannable, keyword-rich, and built for local buyer psychology.

### 💬 Negotiation Buffer Calculator
Shows exactly how much room to leave in your list price so you can accept offers without collapsing below your target. Buyers expect to negotiate. ListingLens prices that in from the start.

### ⚡ Fast-Sale Optimizer
A strategy toggle that shifts your output toward liquidity. When you need to move something today — not next week — fast-sale mode recalibrates your price for speed over margin.

### 🗄️ Valuation Vault
A local history of every item you've priced. Save valuations, mark items as sold, duplicate prior entries, and build a personal pricing reference library. The more you use it, the more useful it becomes.

### 🧪 Listing Trade Simulator *(Premium)*
Test pricing variations before you commit. Compare current vs. modified scenarios. See predicted sell speed, score deltas, key gains, and key risks. Turn ListingLens from a calculator into a strategy tool.

### 🎓 Replayable Walkthrough System
A 7–9 step onboarding tutorial that teaches the app from the ground up. Replayable at any time. Pro users get enhanced depth, advanced optimization tips, and richer listing examples.

### 🔔 Custom Notification System
No native browser alerts. Every confirmation, warning, save, and error uses a smooth, branded in-app notification system. Destructive actions require double confirmation — with clear visual cues to prevent accidents.

### 🔑 Local Premium Licensing *(No Backend Required)*
Unlock premium features with a one-time license key. Validation is fully client-side. No account. No server. No subscription friction. Enter your key, unlock instantly, stay unlocked.

## ⚡ Quick Start

**Requirements:** Node.js 18+ · npm or yarn

```bash
# 1. Clone the repository
git clone https://github.com/TSCREATES-copyright/listinglens.git
cd listinglens

# 2. Install dependencies
npm install

# 3. Start the development server
npm run dev

# 4. Open in your browser
# → http://localhost:5173
```

That's it. No environment variables. No API keys. No backend setup. The app is live.

```bash
# Build for production
npm run build

# Preview the production build
npm run preview
```

The production build is a static site. Drop it on any CDN, Netlify, Vercel, or GitHub Pages.

## 🔁 The Core Workflow

ListingLens is designed to become part of your **selling ritual** — not a one-time tool.

```
BEFORE POSTING AN ITEM
  Open ListingLens → Select category → Enter details
  → Get price → Copy title → Post listing

WHEN A BUYER ASKS "WHAT'S YOUR LOWEST?"
  Open saved valuation → Check negotiation buffer
  → Know your floor before you reply

AFTER SOURCING INVENTORY
  Price each item while you're still at the sourcing location
  → Know your margin before you buy

WHEN COMPARING SIMILAR ITEMS
  Duplicate valuation → Adjust inputs → Compare outputs
  → Pick the better listing strategy

AFTER A SALE
  Mark item as sold → Add actual sale price note
  → Build your personal pricing reference library
```

The more items you price, the more valuable ListingLens becomes. Your history vault becomes a personal benchmark database — calibrated to your categories, your brands, your market.

## 💬 Pricing Decision Engine — In Depth

### Category Anchor System

Each category/subcategory has a **static local resale anchor** — not a live market price, but a stable deterministic baseline built from resale heuristics.

```typescript
// Example anchor structure
{ category: 'electronics', subcategory: 'wireless-headphones', basePrice: 65, volatility: 0.15, demandTier: 'high' }
{ category: 'furniture',   subcategory: 'office-chair',        basePrice: 75, volatility: 0.20, demandTier: 'medium' }
{ category: 'tools',       subcategory: 'cordless-drill',      basePrice: 55, volatility: 0.12, demandTier: 'high' }
{ category: 'baby',        subcategory: 'stroller',            basePrice: 80, volatility: 0.18, demandTier: 'medium' }
{ category: 'gaming',      subcategory: 'console',             basePrice: 95, volatility: 0.10, demandTier: 'high' }
```

### Condition Multipliers

| Condition | Multiplier Range |
|---|---|
| New / Open Box | 1.00 – 1.10 |
| Like New | 0.90 – 0.97 |
| Excellent | 0.80 – 0.90 |
| Good | 0.65 – 0.78 |
| Fair | 0.45 – 0.60 |
| Parts / As-Is | 0.15 – 0.35 |

### Brand Desirability Weights

| Tier | Brands | Weight |
|---|---|---|
| Tier A | Apple, Sony, Bose, Nike, DeWalt, Dyson | 1.15 – 1.35 |
| Tier B | Samsung, Cuisinart, Black+Decker, Adidas | 1.00 – 1.10 |
| Tier C | Off-brand, local brands, store brands | 0.85 – 0.95 |
| Generic | Unbranded, unknown | 0.70 – 0.85 |

### Age Decay Curve

| Age | Electronics | Furniture | Tools |
|---|---|---|---|
| 0–1 yr | -2% | -1% | -1% |
| 1–3 yrs | -12% | -6% | -5% |
| 3–5 yrs | -22% | -14% | -10% |
| 5+ yrs | -35% | -20% | -15% |

### Accessory Bonus

| Accessory | Value Increase |
|---|---|
| Original box | +5% |
| Charger / cable | +3% |

## ⚙️ System Architecture Principles

### Rule 1: Logic Lives Outside React
All pricing logic, validation, and data transformations live in `systems/` as pure TypeScript functions. React only renders state and collects input. This keeps the engine testable, deterministic, and fast.

### Rule 2: Deterministic by Design
Same inputs always produce the same outputs. No AI. No external APIs. No randomness. No drift. If something changes in the output, a human changed the rules intentionally.

### Rule 3: Local-First Storage
- **LocalStorage** → small preferences, settings, feature flags, last-used state
- **IndexedDB** → structured records, full history, simulations, archived items
- Memory cache → active computation results and recently accessed records

### Rule 4: No Component Calculates Prices
Components display results. They do not compute them. Every pricing calculation routes through `CoreEngine.ts`. Enforced by architecture.

### Rule 5: Destructive Actions Require Double Confirmation
Delete, reset, and clear actions require two steps: a warning state and a final confirmation. The UI makes the cost of the action unmistakably clear before proceeding.

## 🧠 Engine Decision Logic — Category Branching

The CoreEngine doesn't apply a flat multiplier to every item. It branches by **category behavior class** — because a gaming console and a couch respond very differently to the same inputs.

### High-Demand Items
*Examples: gaming consoles, wireless headphones, power tools, small appliances*

```
Behavior:
  → Narrower confidence range (more predictable market)
  → Smaller negotiation spread (buyers are more decisive)
  → Lower age penalty (demand absorbs depreciation)
  → Faster expected sell speed at correct price

Engine adjustment:
  negotiationCushion × 0.85
  confidenceBonus    + 8
  agePenalty         × 0.80
```

### Bulky / Local-Pickup-Sensitive Items
*Examples: furniture, treadmills, large appliances, outdoor equipment*

```
Behavior:
  → Stronger urgency discount (pickup friction reduces buyer pool)
  → Heavier condition sensitivity (buyers inspect in person)
  → Larger negotiation room (buyers expect to talk price)
  → Slower expected sell speed even at fair price

Engine adjustment:
  urgencyFactor      × 1.25
  conditionWeight    × 1.15
  negotiationCushion × 1.30
  confidencePenalty  + 6
```

### Commoditized Items
*Examples: generic accessories, common cables, basic tools, household basics*

```
Behavior:
  → Lower confidence ceiling (many identical listings)
  → Sharper discounting required to stand out
  → Tighter profit margin (buyers shop by price alone)
  → Title quality matters more than condition

Engine adjustment:
  basePrice          × 0.85
  confidenceCeiling  = 65
  negotiationCushion × 0.70
```

### Premium Branded Items
*Examples: Apple products, Dyson, premium audio, designer goods*

```
Behavior:
  → Stronger brand weight (brand trust holds resale value)
  → Narrower confidence range (well-understood market)
  → Better title signals (model number matters critically)
  → Higher price ceiling before "too ambitious" warning

Engine adjustment:
  brandWeight        × 1.30
  priceCeiling       + 25%
  confidenceBonus    + 12
  titlePriority      = HIGH
```

## 📐 Confidence Score — Full Specification

The confidence score is a measure of **model reliability** — not item desirability. It answers: *"How much should the user trust this output?"*

### Scoring Formula

```typescript
let confidence = 50; // baseline

// Positive signals
if (categoryIsCommon)        confidence += 12;
if (brandIsRecognized)       confidence += 10;
if (conditionIsExplicit)     confidence += 8;
if (accessoriesSpecified)    confidence += 6;
if (ageBandProvided)         confidence += 5;
if (subcategorySelected)     confidence += 7;
if (strategyModeSet)         confidence += 3;

// Negative signals
if (categoryIsNiche)         confidence -= 15;
if (brandIsUnknown)          confidence -= 8;
if (inputsAreVague)          confidence -= 12;
if (conditionIsSubjective)   confidence -= 6;
if (itemIsCollectible)       confidence -= 10;

// Clamp to valid range
confidence = clamp(confidence, 0, 100);
```

### Confidence Bands

| Score | Label | Meaning |
|---|---|---|
| 85–100 | High Confidence | Output is tightly calibrated. Trust the price. |
| 65–84 | Good Confidence | Output is reliable. Minor variance possible. |
| 45–64 | Moderate Confidence | Add more detail to improve accuracy. |
| 25–44 | Low Confidence | Wide price range. Use as rough guidance only. |
| 0–24 | Very Low Confidence | Item is too unusual or vague to price reliably. |

### Warning Triggers

```
⚠️  "Too Ambitious"
    → Fires when user-set price > 120% of engine list price
    → Fires when urgency is high AND price is still above range
    → Intensifies when confidence is below 50

⚠️  "Low Confidence"
    → Fires when confidence score < 45
    → Prompts user to add brand, condition, or accessory details

⚠️  "Fast-Sale Mode Recommended"
    → Fires when urgency is 'urgent' AND list price > fast-sale price × 1.15
    → Tells user they're pricing against their stated urgency

⚠️  "Pricing Gap Too Wide"
    → Fires when list price and fast-sale price differ by more than 40%
    → Suggests narrowing strategy for cleaner buyer experience

ℹ️  "Accessories Add Significant Value"
    → Info badge when accessory bonus exceeds 8%
    → Reminds user to highlight accessories prominently in title
```

```typescript
// gamification/simulationEngine.ts

export function computeScoreDelta(
  baseRecord: ValuationRecord,
  modifiedInputs: Partial<ValuationRecord>
): number {
  const baseScore  = computeListingScore(baseRecord);
  const modifiedRecord = { ...baseRecord, ...modifiedInputs };
  const modScore   = computeListingScore(modifiedRecord);
  return modScore - baseScore;
}

function computeListingScore(record: ValuationRecord): number {
  let score = 50;
  score += record.confidence * 0.3;
  if (record.urgency === 'urgent')  score -= 10;
  if (record.urgency === 'relaxed') score += 5;
  if (record.strategy === 'profit') score += 8;
  if (record.strategy === 'speed')  score -= 5;
  if (record.accessories.length > 2) score += 8;
  return clamp(score, 0, 100);
}
```

## 🔬 Hooks Reference

### `useComputation`
Triggers engine recalculation whenever form inputs change. Returns memoized pricing output.

```typescript
const { result, isComputing, warnings } = useComputation(formInputs);
// result: ValuationOutput | null
// isComputing: boolean (always near-instant)
// warnings: WarningMessage[]
```

### `useAppState`
Global state access across all panels.

```typescript
const {
  inputs, setInputs,
  result, savedItems,
  activeMode, setActiveMode,
  premiumUnlocked
} = useAppState();
```

### `useToast`
Triggers in-app notifications.

```typescript
const { toast } = useToast();

toast.success("Item saved to vault");
toast.warning("Confidence is low — add more details");
toast.error("Invalid license key");
toast.info("Price is near the upper heuristic limit");
```

### `useConfirmAction`
Wraps destructive actions in a double-confirmation flow.

```typescript
const { confirm } = useConfirmAction();

const handleDelete = () => {
  confirm({
    title: "Delete this valuation?",
    body: "This cannot be undone.",
    icon: "🗑️",
    onConfirm: () => StorageManager.deleteValuation(id),
  });
};
```

### `useLicense`
Reads and updates premium unlock state.

```typescript
const { isPremium, unlock, licenseKey } = useLicense();

// Unlock premium
unlock("LENS-A1B2-C3D4"); // → validates key → stores locally → isPremium = true
```

### `useSimulator`
Manages simulator state and scenario comparison.

```typescript
const {
  scenarios, activeScenario,
  createScenario, applyPreset,
  compareScenarios, saveScenario,
} = useSimulator(baseRecord);
```

### `useIndexedDB`
Structured IndexedDB access with async read/write.

```typescript
const { getAll, put, deleteById } = useIndexedDB<ValuationRecord>('valuations');

const allItems = await getAll();
await put(newRecord);
await deleteById(record.id);
```

## 🤝 Contributing

Contributions are welcome. ListingLens is designed to stay maintainable by a small team — keep that in mind.

```bash
# Fork the repository
git fork https://github.com/TSCREATES-copyright/listinglens.git

# Create a feature branch
git checkout -b feature/your-feature-name

# Make your changes
# All pricing logic must stay in systems/
# All UI must stay declarative — no calculations in components

# Commit with a clear message
git commit -m "feat: add garage cleanout planner mode"

# Push and open a pull request
git push origin feature/your-feature-name
```

### Contribution Guidelines

- **New pricing logic** → add to `systems/` as a pure function
- **New UI component** → add to `components/` with no embedded business logic
- **New category data** → extend the anchor tables with a source reference
- **Bug fixes** → include a test case that demonstrates the fix
- **New features** → open an issue first to discuss scope and approach

## 🏷️ Naming Convention Reference

### Listing Title Formula

```
[Brand] + [Item Type] + [Key Feature] + [Condition] + [Accessories/Context]
```

### Examples

```
Sony WH-1000XM4 Wireless Headphones · Excellent · Charger + Original Box
DeWalt 20V Drill Set · Like New · 2 Batteries Included · Local Pickup
Herman Miller Aeron Chair · Good Condition · Full Adjustments · Pickup Only
Apple iPad Air 5th Gen · Excellent · Smart Cover + USB-C Cable Included
Vitamix 5200 Blender · Like New · Original Box · Barely Used
```

### What Makes a Title Perform

- **Brand first** — buyers search brand names
- **Exact model** — specificity builds trust
- **Condition** — buyers filter and scan for this
- **Accessories** — dramatically increases clicks
- **Local cues** — "local pickup" signals legitimacy
